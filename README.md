# Интерактивная карта Linux

Учебное веб-приложение — упрощённая визуальная модель ядра Linux, userspace-компонентов
и инструментов наблюдения. Граф показывает подсистемы (hardware → kernel → userspace →
observability), связи между ними и типовую причинно-следственную динамику.

> ⚠️ Это **не эмулятор**. Приложение полностью статическое, работает только в браузере и
> **не обращается** к реальным `/proc`, `/sys`, kernel API и не выполняет системные команды.

## Возможности

- Интерактивный граф подсистем Linux на [React Flow](https://reactflow.dev/) — канвас в центре интерфейса.
- Клик по узлу открывает панель описания: за что отвечает, входящие/исходящие связи
  (полными фразами), параметры, файлы, команды, поведение под нагрузкой.
- Сценарии с пошаговой анимацией: задействованные узлы **возвышаются** и подсвечиваются,
  а по активным связям бежит «пакет» — как анимация трафика в Cisco Packet Tracer.
- Режим «Объяснить путь» между двумя узлами (BFS по графу связей).
- Метрики (CPU, память, I/O, сеть, OOM и др.), меняющиеся по шагам сценария.
- Импорт snapshot JSON (метрики, службы, sysctl) — метрики обновляются, связанные узлы подсвечиваются.
- Справочник sysctl / cgroup v2 параметров и диагностические карточки — в модалке **«Справка»**.

## Быстрый старт

Нужен [Node.js](https://nodejs.org/) 18+ (рекомендуется 20). Пакетный менеджер — `pnpm`
(есть lock-файл) или `npm`.

> Нет Node на машине (частый случай на «чистом» Linux-сервере)? Пропустите варианты 1–2
> и используйте **Docker (вариант 3)** — он не требует установленного Node. Скрипт
> `start.sh` тоже это подскажет, если не найдёт npm/pnpm.
>
> Установить Node на Debian/Ubuntu/Astra: `sudo apt update && sudo apt install -y nodejs npm`

### Вариант 1. Ярлык (самый простой)

- **Windows:** двойной клик по `start.cmd`
- **Linux / macOS:**
  ```bash
  chmod +x start.sh   # один раз
  ./start.sh
  ```

Скрипт сам поставит зависимости при первом запуске, откроет браузер и поднимет dev-сервер.

### Вариант 2. Вручную

```bash
pnpm install   # или: npm install
pnpm run dev   # или: npm run dev
```

Откройте адрес, который покажет Vite (по умолчанию http://localhost:5173).

### Вариант 3. Docker (любой Linux, без установки Node)

```bash
docker compose up --build
```

Приложение будет доступно на http://localhost:8080.

Либо вручную:

```bash
docker build -t linux-kernel-visual-model .
docker run --rm -p 8080:80 linux-kernel-visual-model
```

Образ многостадийный: Node собирает статику (`vite build`), затем nginx раздаёт её
с SPA-fallback. Итоговый образ содержит только nginx + собранный бандл.

## Скрипты

| Команда | Действие |
| --- | --- |
| `pnpm run dev` | dev-сервер с HMR |
| `pnpm run build` | проверка типов (`tsc --noEmit`) и продакшн-сборка в `dist/` |
| `pnpm run preview` | локальный предпросмотр собранного бандла |

## Управление интерфейсом

- **Клик по узлу** — детали в панели слева.
- **Панель описания** сворачивается кнопкой в её шапке (состояние запоминается).
- **Поиск** вверху — по узлам, параметрам, сценариям и проблемам.
- **Выбор сценария + ▶** — запуск пошаговой имитации; **⏸** пауза, **↺** сброс.
- **Timeline** внизу — переход по шагам; сворачивается.
- **Импорт** — загрузка snapshot JSON.
- **Справка (?)** — параметры, диагностика и заметки о модели в одной модалке.

## Структура проекта

```
src/
  App.tsx              — состояние и композиция интерфейса
  components/
    LinuxMap.tsx       — граф (кастомные узлы + рёбра с анимацией пакета)
    Inspector.tsx      — панель описания узла
    TopBar.tsx         — поиск, сценарии, импорт, справка
    HelpModal.tsx      — параметры / диагностика / о модели
    Timeline.tsx, MetricsPanel.tsx, ScenarioPanel.tsx, LearningPath.tsx
    ErrorBoundary.tsx  — защита от «схлопывания» UI при ошибке рендера
  data/                — узлы, связи, параметры, сценарии, проблемы, метрики
  utils/model.ts       — индексы, поиск, BFS-путь, парсинг snapshot
  types.ts, styles.css
```

## Расширение модели

**Узел** — `src/data/nodes.ts`, helper `node(...)`: `id`, название, слой
(`hardware` / `kernel` / `userspace` / `observability`), категория, описание, параметры,
команды, файлы, типовые проблемы, поведение под нагрузкой. Связи — в `src/data/edges.ts`.

**Параметр** — `src/data/parameters.ts` (`sysctlParameter(...)` / `cgroupParameter(...)`):
имя, подсистема, путь, тип, пример, безопасный диапазон, что меняет, на что влияет,
связанные узлы/сценарии, команды просмотра/изменения (как справка), риск.

**Сценарий** — `src/data/scenarios.ts` (`Scenario`): шаги со временем, заголовком,
описанием, затронутыми узлами, изменёнными параметрами, командами и частичным `MetricSnapshot`.

## Импорт snapshot JSON

Кнопка «Импорт» в верхней панели принимает JSON вида:

```json
{
  "hostname": "test-host",
  "kernel": "6.1.0",
  "metrics": {
    "memTotalMb": 16000,
    "memAvailableMb": 4200,
    "swapUsedMb": 512,
    "loadAverage": [1.2, 1.8, 2.1],
    "processCount": 248
  },
  "services": [
    { "name": "nginx.service", "state": "active" },
    { "name": "app.service", "state": "failed" }
  ],
  "sysctl": {
    "vm.swappiness": "60",
    "vm.dirty_ratio": "20"
  }
}
```

Ввод валидируется; импорт идемпотентен (повторная загрузка того же файла даёт тот же результат).
Максимальный размер файла — 2 МБ.

## Ограничения модели

- Это не эмулятор ядра; сценарии и метрики искусственные.
- Связи показывают типовые причинно-следственные цепочки, а не все code paths ядра.
- Состояние systemd service — это состояние unit, а не состояние ядра.
- `/proc` и `/sys` показаны как интерфейсы наблюдения/управления, а не обычные файлы.
- Backend отсутствует, данные не покидают браузер.
